Personal Portfolio Tracker - Architecture (Free APIs Only)

Version 1.1 | Date: 2026-03-07 | Stack: JavaScript Frontend + Spring Boot Backend + Oracle DB

Policy: Free APIs only Single-user portfolio Auto-sync first, import fallback

1. Goal and Constraints

Build a portfolio dashboard for Debt MF, Equity MF, EPF, Indian stocks (HDFC Securities), US stocks (INDmoney), Bitcoin (CoinDCX), and FDs, with maximum automated updates using only free data sources.

4.1 Backend modules

  • portfolio-core: accounts, instruments, transactions, holdings.
  • connector-amfi: download + parse NAV text feed.
  • connector-alpha: daily stock price pulls within free quota.
  • connector-coindcx: BTC market ticker + optional authenticated balances.
  • connector-fx: USD-INR rate fetch for US holdings valuation.
  • import-engine: parse CSV/statement files and normalize ledger entries.

5. Oracle Data Model (Core)

  • ACCOUNT (account metadata, source mode: API/IMPORT/MANUAL)
  • INSTRUMENT (symbol/ISIN/asset class/currency)
  • TXN_LEDGER (buy/sell/sip/dividend/interest/fd maturity)
  • HOLDING_POSITION (daily derived positions)
  • PRICE_POINT (source-tagged prices)
  • FX_RATE (USD-INR/EUR-INR reference rates)
  • VALUATION_SNAPSHOT (market value + unrealized PnL)
  • SYNC_JOB and RAW_PAYLOAD (audit + replay)
  • RECON_ISSUE (mismatch queue)

8. Free API Governance Rules

  1. Always cache the latest successful price/NAV per symbol for 24h fallback.
  2. Batch symbol calls to minimize API credits.
  3. Store source_name, fetched_at, and payload hash for every ingest.
  4. If free API limit is hit, mark data as STALE and show warning banner in dashboard.
  5. Never block portfolio view because one connector fails.
Keep the system accurate by separating: price sync (API) and holdings sync (import/manual where required).
